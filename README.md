## about

I'm a backend engineer with a primary focus on **Go**. I design systems that are fast by default, maintainable by design, and built to handle real load — not just happy path.

I care about the things that actually matter in production: proper error handling, connection pooling, graceful shutdowns, sane retry logic, and services that don't wake you up at 3am.

From database schema to deployment — I own the full backend lifecycle.
