## about

Backend engineer who chose **Go** and never looked back. I build services that handle real traffic — not just pass unit tests. My default mode is thinking about what breaks at scale before writing the first line.

I obsess over the invisible layer: connection lifetimes, query plans, goroutine leaks, retry storms, graceful degradation. The stuff users never see but always feel. Production is the real test — and I've been studying it hard.

Currently channeling that into **Codeway**, where I've taken 1000+ students from zero to actually understanding how systems work. Teaching sharpens thinking. You can't fake understanding when someone asks *why*.

From database schema to Dockerfile to deployment — I own the full backend lifecycle, alone or in a team.
