## About Me

I'm a backend engineer specializing in **Go**. I build production-grade services with a focus on performance, reliability, and clean architecture — systems designed to handle real load and stay maintainable as they grow.

My work lives at the intersection of correctness and efficiency: proper concurrency patterns, optimized database access, robust error handling, and services that degrade gracefully under pressure rather than fall apart.

I approach backend development as an engineering discipline — every decision from schema design to deployment has a reason, and I care deeply about getting those decisions right.
